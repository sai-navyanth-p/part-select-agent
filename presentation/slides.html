PartSelect AI Chat Agent

A multi-agent customer support system for refrigerator and dishwasher replacement parts - powered by GPT-4o with streaming, hybrid search, and domain-specialized agents.

Built by Sai Navyanth

The Problem

Why does PartSelect need this?

Technology

Tech Stack

LLM

GPT-4o + 4o-mini

4o for specialist responses & tool use. 4o-mini for fast routing & memory summarization. ~60% cost reduction.

Backend

FastAPI + SSE

Async Python framework with Server-Sent Events for real-time token streaming.

Frontend

Next.js + React

TypeScript, SSE consumption, typewriter rendering, PartSelect-branded UI.

Relational DB

SQLite

28 parts, 20 models, 86 compatibility links, guides, orders. Zero-infra.

Vector DB

ChromaDB

Semantic search over products & guides. Local embeddings via all-MiniLM-L6-v2.

Scraper

Playwright

Stealth mode scraper for PartSelect.com. Visible browser to bypass anti-bot.

Key Features

What Makes This Different

üîÄ

Smart Routing

GPT-4o-mini classifies intent in ~200ms at < $0.001/query. Each specialist only sees its own tools - no confusion.

üîç

Hybrid Search

SQL catches exact matches (PS numbers, brands). Semantic search catches fuzzy queries ("that thing that makes ice"). Combined results.

‚ö°

Real-Time Streaming

Tokens stream via SSE with typewriter effect. Specialist badge shows which agent is responding. Sync fallback if stream fails.

üõ°

Guardrails

Pre-LLM validation: length limits, prompt injection detection, off-topic filtering. No API cost, instant response.

üß†

Conversation Memory

Sliding window keeps last 10 messages. Older messages summarized via GPT-4o-mini. Token counting with tiktoken.

üìä

Rich Product Cards

Interactive cards with pricing, ratings, stock status, installation difficulty, and direct links to PartSelect.com.
